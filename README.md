# transcribe-video

동영상 강의나 강연을 텍스트로 옮겨, 요약과 지식화([LLM Wiki](https://x.com/karpathy/status/2039805659525644595))의 입력으로 쓰려고 만들었습니다. 상용 STT의 비용과 업로드 제약 없이 내 PC GPU만으로 강의 영상 수십 개를 한 번에 글로 옮기고, 전사는 [openai/whisper](https://github.com/openai/whisper)보다 효율적인 [faster-whisper](https://github.com/SYSTRAN/faster-whisper)로 처리합니다. 한국어에서 자주 나오는 Whisper 환각은 직접 설계한 후처리로 걸러냅니다.

## 왜 만들었나

동영상 강의나 유튜브 강연을 텍스트로 만들어 검색하고 정리하고 싶었습니다. 로컬 LLM과 개인 GPU 성능이 올라오고 프론티어 LLM이 긴 스크립트에서 의미를 뽑아내면서, 추출된 스크립트는 단순 기록을 넘어 요약과 지식화(LLM Wiki)의 입력으로 쓸모가 커졌습니다. 그런데 막상 영상을 대량으로 글로 바꿀 쓸 만한 방법은 마땅치 않았습니다.

상용 STT 서비스를 써보며 불편했던 점은:

- **비용과 업로드**: 영상이 길고 많을수록 비용이 커지고, 비음성 구간을 편집해서 올려야 비용을 아낄 수 있었습니다.
- **한국어 환각**: 같은 whisper 계열 서비스도 무음이나 잡음 구간에서 없는 문장을 만들어냈습니다.
- **운영 번거로움**: 유튜브 영상이나 여러 파일을 미리 작업 목록으로 등록해두기가 번거로웠습니다.

그래서 직접 만들기로 했습니다. 개인 PC GPU로 무료로 돌리고, 한국어 품질을 직접 손보고, 일괄 처리와 이어하기, 자막 없는 유튜브 영상까지 되는 도구입니다.

## 어떻게 해결했나

Whisper 호출 자체는 쉬웠습니다. 어려운 건 그 다음, 실사용에서 부딪힌 문제를 하나씩 설계로 푸는 일이었습니다.

### 1. 한국어 환각 걸러내기

처음 결과물에는 무음 구간의 자막 크레딧, 멀리 떨어져 반복되는 같은 말처럼 사람이 말하지 않은 문장이 섞였습니다. 특정 단어를 막으면 진짜 발화까지 지울 위험이 있어, 성격이 다른 네 겹의 필터를 겹쳤습니다.

- **지표 기반**: avg_logprob, no_speech_prob, compression_ratio가 극단적이거나 글자 밀도가 낮은 세그먼트를 버립니다.
- **결정론적 반복 탐지**: 같은 텍스트의 반복 간격이 모두 수초 이상이면 환각으로 봅니다. 같은 입력에 같은 출력을 내는 환각의 특성을 추임새 반복과 구분합니다.
- **관측 문구 정확 일치**: 실제로 본 환각 문자열만 막고, 정규식으로 일반화하지 않습니다.
- **Whisper 내장 탐지**: word_timestamps와 hallucination_silence_threshold를 켭니다.

여기서 "더 잡되 오삭제를 감수할지, 보수적으로 진짜 발화를 지킬지"를 명시적으로 선택해야 했습니다. 오삭제가 더 치명적이라고 보고, 관측된 사례만 막는 보수적 설계를 택했습니다.

### 2. 무음 구간 건너뛰기

무음과 잡음은 그대로 넣으면 환각의 입력이 되고 시간도 낭비됐습니다. 0.5초 단위로 RMS 에너지, zero-crossing rate, 스펙트럼 평탄도, 음성 대역 비율을 계산해 음성 구간만 추리는 분석기를 직접 구현하고, 그 구간만 Whisper에 넘겼습니다. 분석 구간이 오디오 끝을 넘으면 Whisper가 무한 루프에 빠지는 버그는 마지막 구간 끝을 살짝 당겨 막았습니다.

### 3. 한글 경로 깨짐

- ffmpeg가 한글 경로를 직접 열지 못해, 영상 파일을 Python에서 열어 ffmpeg 표준 입력으로 흘려보냈습니다.
- yt-dlp는 출력이 파이프로 연결되면 콘솔 코드페이지(cp949)로 인코딩해 경로가 깨졌습니다. `--encoding UTF-8`로 출력 인코딩을 맞춰 해결했습니다.

### 4. 실행본 빌드의 네이티브 크래시

자체 포함 실행본(PyInstaller onedir)으로 배포하며 두 번 크래시를 만났습니다.

- 요청이 끝날 때 GC가 모델을 해제하다 "Fatal Python error: Aborted"로 죽었습니다. 모델을 모듈 전역에 하나만 캐시해 해제 시점을 없애고 재로딩 비용도 줄였습니다.
- 음성인식 엔진과 PyTorch가 각자 OpenMP 런타임을 들고 있어 중복 로드되면 종료 시점에 네이티브 abort가 났습니다. 진입 전 환경변수로 고정해 막았습니다.

### 5. 진행률 실시간 스트리밍

faster-whisper의 transcribe는 generator를 돌려주고, 그 반복이 곧 추론입니다. 동기 반복이 비동기 서버를 막지 않도록 추론은 워커 스레드에서 돌리고, 결과를 asyncio 큐로 넘겨 WebSocket으로 즉시 전송했습니다. 추론과 전송을 분리해 연결이 끊겨도 전사는 끝까지 가도록 했습니다.

### 6. 빌드와 실행을 한 번으로 캡슐화

루트의 `start.bat` 하나가 내용 해시로 소스 변경을 확인합니다. 바뀌었으면 새로 빌드하고, 그대로면 마지막 빌드를 실행한 뒤 서버를 띄우고 브라우저를 엽니다. 같은 코드는 다시 빌드하지 않습니다.

## 결과

- **로컬 폴더 일괄 전사**와 **유튜브 다운로드 전사**(단일 영상과 재생목록)
- **GPU 가속**: GPU가 있으면 float16, 없으면 CPU int8로 자동 전환
- **출력**: 원본 옆에 `.txt`, 선택 시 `.srt` 자막
- **실시간 진행률**과 **이어하기**(이미 처리한 파일은 건너뜀)
- **PyInstaller 실행본 패키징**과, 변경 시 자동 빌드 후 실행하는 **start.bat 진입점**

기술 스택은 Python, FastAPI, WebSocket, faster-whisper(CTranslate2), PyTorch(CUDA), NumPy, ffmpeg, yt-dlp, PyInstaller입니다.

## 빠른 시작

`uv`가 설치돼 있고 `ffmpeg`가 PATH에 등록돼 있어야 합니다. 준비되면 루트의 `start.bat`을 더블클릭합니다. 처음 실행하거나 소스가 바뀌었으면 자동으로 빌드하고, 기존 빌드가 유효하면 마지막 빌드를 실행한 뒤 서버를 띄우고 브라우저(http://localhost:8765)를 엽니다. 최초 실행은 환경 준비와 빌드로 시간이 걸립니다.

## 구조

문서와 진입점을 루트에 두고, 프로젝트 본체를 한 겹 아래 `transcribe-video/`에 모았습니다. 설계와 모듈별 상세는 [PROJECT.md](PROJECT.md)에 있습니다.

```
transcribe-video/            repo 루트
├── start.bat  start.ps1     사용자용 진입점 (변경 시 빌드 후 실행)
├── README.md  PROJECT.md  images/  docs/
└── transcribe-video/        프로젝트 (소스/빌드/venv)
    ├── pyproject.toml  uv.lock
    └── src/  tests/  scripts/  bin/
```

## 개발

```
cd transcribe-video
uv venv
uv sync --group dev
```

- 서버: `uv run python src/server.py` (http://localhost:8765)
- CLI: `uv run python src/transcribe_video.py <영상 또는 폴더> [--batch] [--srt]`
- 테스트: `uv run pytest`

## 회고
